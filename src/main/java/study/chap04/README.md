### 4. TDD, 기능 명세, 설계

#### 4.1. 기능 명세

- 기능은 크게 두 가지: **입력, 결과**

```
    입력: 아이디와 암호
    결과: 아이디와 암호가 일치하면 성공, 아니면 실패
```

- 결과: 리턴 값, 익셉션, 시스템의 상태 변경(DB 등)
  - **시스템의 상태 변경이 결과에 포함된다면 테스트는 변경 대상에 접근해 결과를 확인해야 함**
- 기능 명세의 입력과 결과를 코드에 반영하는 과정에서 기능의 이름, 파라미터, 리턴 타입 등이 결정. -> 기능에 대한 설계
